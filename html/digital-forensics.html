Advanced Digital Forensics

Week 1: Foundations of Digital Forensics

  1. Day 1: Overview of DFIR lifecycle, chain of custody, roles and responsibilities.
  2. Day 2: Introduction to legal frame and forensic standards (NIST 800-86, ISO 27037).
  3. Day 3: Criminal Investigation, Digital Crime Scene and Case study: timeline from detection to reporting.
  4. Day 4: Lab – Build forensic lab environment (SIFT, Autopsy).

Tools: SIFT, Autopsy, FTK Imager | Assessment: Documentation accuracy, environment readiness.

Week 2: Evidence Acquisition & Verification

  1. Day 1: Evidence types, volatility, imaging theory.
  2. Day 2: Disk and memory imaging with FTK, dc3dd.
  3. Day 3: RAM capture and live system acquisition challenges.
  4. Day 4: Lab – Verify image integrity using hash values.

Tools: FTK Imager, dc3dd, Magnet RAM Capture | Assessment: Correct acquisition & hash validation.

Week 3: File System & Data Recovery

  1. Day 1: File system structure and metadata (NTFS/FAT).
  2. Day 2: Data carving and recovery techniques.
  3. Day 3: Detecting hidden partitions and encrypted containers.
  4. Day 4: Lab – Recover deleted files using Autopsy.

Tools: SleuthKit, X-Ways, TestDisk | Assessment: File recovery success rate.

Week 4: Windows Artifacts Analysis

  1. Day 1: Registry forensics and persistence.
  2. Day 2: Event logs, Prefetch, Amcache analysis.
  3. Day 3: LNK and user activity correlation.
  4. Day 4: Lab – Timeline reconstruction using Plaso.

Tools: RegRipper, RECmd, Plaso | Assessment: Timeline completeness & accuracy.

Week 5: Linux & macOS Forensics

  1. Day 1: Linux file and log structure.
  2. Day 2: macOS artifacts (plist, Spotlight).
  3. Day 3: File timestamp and permissions analysis.
  4. Day 4: Lab – Correlate user and system activity logs.

Tools: SleuthKit, log2timeline | Assessment: Log correlation accuracy.

Week 6: Memory Forensics

  1. Day 1: Memory acquisition formats and analysis basics.
  2. Day 2: Process and DLL enumeration.
  3. Day 3: Rootkit and malware detection in RAM.
  4. Day 4: Lab – Identify malicious processes via Volatility.

Tools: Volatility3, Rekall, Redline | Assessment: Memory analysis findings.

Week 7: Network & Web Forensics

  1. Day 1: Packet capture review (PCAP).
  2. Day 2: Web traffic reconstruction (HTTP/DNS).
  3. Day 3: IDS/IPS alert correlation.
  4. Day 4: Lab – Trace intrusion via PCAP and Zeek logs.

Tools: Wireshark, Zeek, Suricata | Assessment: Attack reconstruction clarity.

Week 8: Mobile Forensics

  1. Day 1: Mobile acquisition methods.
  2. Day 2: App and chat data parsing.
  3. Day 3: GPS and metadata extraction.
  4. Day 4: Lab – Analyze Android image with Andriller.

Tools: MOBILedit, Cellebrite demo, Andriller | Assessment: Artifact extraction quality.

Week 9: Cloud & Virtual Forensics

  1. Day 1: Cloud evidence identification (AWS, Azure).
  2. Day 2: Log collection and validation.
  3. Day 3: VM evidence imaging (VMDK, QCOW2).
  4. Day 4: Lab – Investigate AWS CloudTrail data breach.

Tools: AWS CLI, Azure Sentinel | Assessment: Incident reconstruction accuracy.

Week 10: Reporting & Legal Frameworks

  1. Day 1: Legal admissibility of evidence.
  2. Day 2: Report writing best practices.
  3. Day 3: AI-assisted report generation.
  4. Day 4: Lab – Write forensic report and defend findings.

Tools: TheHive, CaseNotes, Gemini API | Assessment: Report clarity & professionalism.

Week 11: Capstone – CTF Challenge

  1. Day 1: Incident briefing & data acquisition.
  2. Day 2: Analysis and artifact correlation.
  3. Day 3: Timeline & reporting.
  4. Day 4: Presentation to instructor panel.

Tools: Full toolchain | Assessment: Final report, oral defense, and tool mastery.

Week 12: IoT & Embedded Forensics

  1. Day 1: IoT evidence sources and challenges.
  2. Day 2: Firmware and flash extraction.
  3. Day 3: Drone and automotive forensics.
  4. Day 4: Lab – Analyze IoT log and firmware image.

Tools: Binwalk, Autopsy, IoT Inspector | Assessment: Firmware evidence recovery.

Week 13: AI & ML in Forensics

  1. Day 1: Intro to ML-based anomaly detection.
  2. Day 2: NLP for phishing and artifact classification.
  3. Day 3: Automated log triage via AI.
  4. Day 4: Lab – Build ML model or use AI tools for log anomaly detection.

Tools: Python, Scikit-learn, TensorFlow | Assessment: Model accuracy and interpretation.
